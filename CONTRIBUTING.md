# Contributing to FinMaster Pro

Thank you for your interest in contributing to FinMaster Pro.

This project prioritizes security, simplicity, and maintainability.

---

## Contribution Guidelines

### You may contribute by:
- Fixing bugs
- Improving UI/UX
- Refactoring code for clarity
- Improving documentation
- Suggesting performance improvements

---

## What NOT to Submit

- Backend services
- Tracking or analytics code
- Hardcoded secrets or credentials
- Features that compromise user privacy
- Automatic background sync without consent

---

## Code Style

- Vanilla JavaScript only
- No frameworks
- Modular structure
- Clear separation of concerns
- Security-first mindset

---

## Pull Requests

- Keep PRs focused and minimal
- Explain **why** a change is needed
- Reference related issues if applicable
- Ensure no sensitive data is included

---

## Security Issues

Do not submit security vulnerabilities via public issues.  
Please see `SECURITY.md` for reporting guidelines.

---

## Final Note

FinMaster Pro is designed to remain:
- Offline-first
- Privacy-respecting
- User-controlled

Contributions should align with these principles.
